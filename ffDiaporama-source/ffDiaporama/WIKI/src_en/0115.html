This dialog allows you to render the project as a video file.

This dialog has different appearances depending on whether you clicked the Device type, Lossless, Advanced or Soundtrack button.

Description of the dialog box

In device mode

Destination file Indicate here the name of the destination file.
Click on this button to select graphically a file and a directory.
Device type Allows you to select a device type.
Model Allows you to select a device model.
Rendering The rendering can be made on the whole project or only on a part.
This choice is made:
  • By checking the option All slides to render the video of the whole project
  • Or by checking the option From and then entering the numbers of the start and end slides to generate a video of only a part of the project.
Sound (include sound in the video) If this box is checked then the sound track will be included in the video
Language Defines the value of the Language field.
This value is used by players to indicate in which language is the audio track. This value is on 3 characters and follows this recommendation: ISO 639
Join thumbnail to rendered file If this box is checked then a thumbnail file will be created with video.
For more information on joined files, see File attachments
Join XBMC .nfo file to rendered file If this box is checked then a XBMC .nfo file will be created with video.
For more information on joined files, see File attachments

Render the movie

Once you have selected the device or set the advanced parameters, one click on the OK button starts the process of rendering the video.
At any time during this process, one click on the "Cancel" button stops the process.

Remarks:

Rendering duration:
Depending on the device model or the advanced parameters which you selected, the generation can take quite a long time.
Generally, for a Full-HD 1080p video:
  • On a PC with 4 fast cores (Core i7 for example) : The process will take approximately 2 minutes for each minute of video to be rendered
  • On a PC with 2 cores (Core 2 duo for example) : The process will take approximately 5 minutes for each minute of video to be rendered
This amounts to 2 to 5 hours for a one-hour video.
This duration will be much shorter for a video in VGA resolution. That is why we recommend you to carry out all tests and focusing trials at low resolution, and to render your videos in Full-HD only for the last step.

Variable bitrate:
If variable bitrate is activated, the video compression bitrate could vary from -50% to +125%
  • If the project contains mostly pictures, this option can significantly reduce the size of the video file (up to 80%)
  • By cons, if the project contains mostly videos this option will increase the size of the video file
By default, variable bitrate is activated when duration of videos are less than half of the total duration of the projet.
 

At the dialog bottom

The properties button allows you to open the Project properties dialog. It allows you to adjust the properties before generating the video.
Note: the project properties are saved into the video as TAG (METADATA) which are used by a lot of indexing programs or by Media Center Systems.
